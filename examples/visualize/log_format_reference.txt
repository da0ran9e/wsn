SCENARIO3 VISUAL DEBUG TXT FORMAT (v1)

1) META lines (optional, many lines)
META key=value

2) NODE lines (required)
NODE id=<uint> x=<double> y=<double> cell=<int> leader=<uint> color=<uint>

3) LINK lines (optional)
LINK a=<uint> b=<uint> kind=<string>

4) STEP blocks (optional, for timeline)
STEP t=<double> suspiciousCells=<comma-separated-int> suspiciousNodes=<comma-separated-uint>
ENDSTEP


SCENARIO4 CELL-FORMATION EXTENSION (v2)

Use same META/NODE/LINK from v1, then add these optional sections.

5) CELL lines (optional, cell summary after formation)
CELL id=<int> leader=<uint> size=<uint> members=<comma-separated-uint> color=<uint>

6) FORMEVENT lines (optional, timeline of cell formation)
FORMEVENT t=<double> stage=<string> formedCells=<comma-separated-int> totalCells=<uint> totalNodes=<uint>

7) FORMSNAPSHOT blocks (optional, detailed per-stage cell state)
FORMSNAPSHOT t=<double> stage=<string>
CELL id=<int> leader=<uint> size=<uint> members=<comma-separated-uint> color=<uint>
ENDFORMSNAPSHOT

Notes:
- `stage` examples: precompute, startup-discovery, stable
- `members` can be '-' if empty
- Parser should ignore unknown lines for forward compatibility
