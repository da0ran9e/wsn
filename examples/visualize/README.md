# Scenario3 Network Visualizer

Simple HTML + JS debug visualizer for Scenario3 network simulation. Reads TXT log format to visualize nodes, links, and suspicious region expansion.

## Files

- **index.html**: Main UI with canvas for network visualization
- **app.js**: Log parser and visualization logic
- **styles.css**: Dark-theme styling
- **sample_network_log.txt**: Example TXT log for testing
- **log_format_reference.txt**: Log format documentation
- **network_log.txt**: Actual simulation output (generated by example3)

## Usage

### 1. Run Simulation and Generate Log

```bash
cd /Users/mophan/Github/ns-3-dev-git-ns-3.46
./ns3 run "example3 --gridSize=5 --simTime=7"
```

This generates `network_log.txt` in the project root.

### 2. Copy Log to Visualizer Folder

```bash
cp network_log.txt src/wsn/examples/visualize/
```

### 3. Start Local Server

```bash
cd src/wsn/examples/visualize
python3 -m http.server 8080
```

Then open: http://localhost:8080

Or if already running, change the filename in the UI:
- **Log file input**: Change `sample_network_log.txt` to `network_log.txt`
- Click **Load**

### 4. Navigate Visualization

- **Prev Step / Next Step**: Browse through suspicious region expansion
- **Summary panel**: Shows nodes, links, suspicious cells/nodes
- **Canvas**: 
  - Blue/green/yellow circles = nodes by cell
  - Red circles = nodes in suspicious region
  - White outline = cell leaders
  - Lines = logical neighbor links

## Log Format

```
META key=value
NODE id=<uint> x=<double> y=<double> cell=<int> leader=<uint> color=<uint>
LINK a=<uint> b=<uint> kind=<string>
STEP t=<double> suspiciousCells=<csv> suspiciousNodes=<csv>
ENDSTEP
```

## Code Integration

In scenario3.cc:

1. **Initialize logger** (in example3.cc):
   ```cpp
   InitializeScenario3Visualizer("network_log.txt");
   ```

2. **Log network data** (automatic via OnLogNetworkForVisualization):
   - Outputs all nodes with positions, cells, leaders
   - Outputs logical neighbor links
   - Outputs metadata (gridSize, spacing, radius)

3. **Log suspicious region** (automatic via OnSelectSignalSourceLocation):
   - STEP entries with suspiciousCells and suspiciousNodes

4. **Close logger** (in example3.cc):
   ```cpp
   CloseScenario3Visualizer();
   ```
